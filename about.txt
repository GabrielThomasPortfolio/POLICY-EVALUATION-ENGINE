================================================================================
AI GOVERNANCE & POLICY EVALUATION ENGINE
================================================================================

OVERVIEW
This application is an Enterprise GRC Infrastructure Tool designed to bridge the 
gap between natural-language corporate policies and structured, deterministic 
regulatory mapping. It automates the evaluation of untrusted policy documents 
against global risk management frameworks (ISO 42001, ISO 27001) and statutory 
legislation (EU AI Act, Brazilian AI Act PL 2338, NIST AI RMF).

By leveraging a secure, multi-layered compliance pipeline, the engine drops 
evaluation timelines from weeks to seconds while maintaining strict data 
privacy and zero-trust execution boundaries.

--------------------------------------------------------------------------------
CORE ARCHITECTURE & TECHNICAL SPECIFICATIONS
--------------------------------------------------------------------------------

1. ADVERSARIAL INPUT FIREWALL & DATA SANITIZATION (sanitize.py)
   - Prompt Injection Defense: Implements a regex-based firewall that actively 
     scans for and intercepts malicious structural overrides (e.g., "ignore 
     previous instructions") before they reach the LLM execution context.
   - PII & Proprietary Masking: Automatically detects and redacts personal 
     identifiable information (PII), network paths, and proprietary corporate 
     markers prior to runtime processing to prevent data leakage.

2. SECURE ORCHESTRATION & HYBRID EXECUTION (orchestrate.py)
   - XML Context Fencing: Wraps untrusted policy input inside hard XML tags 
     (<policy_context>) to strictly separate system-level compliance instructions 
     from user-provided data.
   - Live API & Local Fallback: Integrates the official OpenAI SDK for live 
     evaluation (e.g., gpt-4o-mini), with a seamless architectural fallback to 
     a local deterministic simulation engine if cloud endpoints are isolated.

3. DETERMINISTIC OUTPUT GATE (Circuit Breaker)
   - Pydantic Schema Validation: Replaces standard LLM text generation with a 
     strict, type-enforced schema pipeline. The circuit breaker drops any 
     response that hallucinates data, omits mandatory compliance keys, or breaks 
     formatting, ensuring downstream enterprise reporting systems remain uncorrupted.

4. DYNAMIC REGULATORY OVERLAY ENGINE (transform.py)
   - Multi-Framework Crosswalking: Dynamically maps technical engineering controls 
     (like NIST 800-53) directly to overarching statutory laws (e.g., GDPR Article 35, 
     Colorado AI Act).
   - Weighted Semantic Router: Routes policy text to the correct compliance 
     tracks using a custom scoring matrix based on primary framework identifiers, 
     semantic hooks, and context token intersection.

--------------------------------------------------------------------------------
TECH STACK
--------------------------------------------------------------------------------
- Frontend: Streamlit (Python)
- Validation: Pydantic V2
- AI/ML Orchestration: OpenAI Python SDK (Compatible with local Ollama endpoints)
- Data Processing: JSONL Knowledge Bases, RegEx
